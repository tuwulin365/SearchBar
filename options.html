注释

描述：

当您将鼠标悬停在按钮上时，就会显示此信息。

主页 URL：

这将决定您在不输入查询词的情况下进行搜索时将被引导至的页面。请务必包含 URL 中的“http”部分。

与搜索 URL 一样，您可以使用 %h 插入当前网页的主机名（例如，用于 WHOIS 查询），您可以使用 %u 插入当前网页的完整 URL（例如，用于 Internet Archive 搜索），您可以使用高级语法 %{<JavaScript 表达式>}，并且您可以在 URL 的开头添加“i:”以强制其在隐身窗口中打开。

搜索 URL：

通常情况下，当您通过右键单击添加自定义搜索时，SearchBar 会自动检测到这一点。如果无法自动检测到，您可以尝试手动获取搜索 URL：使用测试查询（例如“TEST”）进行搜索，从地址栏复制 URL，并将所有出现的测试查询替换为“%s”。

除了 %s 之外，您还可以使用 %h 插入当前网页的主机名（例如，用于 Google 网站搜索），使用 %u 插入当前网页的完整 URL（例如，用于 Google 缓存搜索）。

对于更高级的替换，您还可以使用 %{<JavaScript 表达式>}。该表达式可以访问变量 s、h 和 u，以及标准的内置 JavaScript 对象和函数，例如 JSON 和 encodeURIComponent。该表达式将使用 static-eval 进行求值。例如，要插入编码后的搜索查询，但将空格替换为下划线，请使用 %{encodeURIComponent(s.replace(/ /g,"_"))}。

对于 POST 请求，您可以使用 ?? 代替 ?。

要强制在隐身窗口中打开 URL，请在 URL 的开头添加“i:”。

图标 URI：

您可能希望使用网站的 favicon 作为按钮图标。它通常通过 <link rel="shortcut icon"> 或 <link rel="icon"> 标签链接到首页源代码的顶部附近。您可以通过“https://www.example.com/favicon.ico”或“https://www.google.com/s2/favicons?domain=www.example.com”访问它，其中“www.example.com”应替换为网站的主机名。

或者，您可以使用“上传”功能将任何图像转换为数据 URI。

热键：

如果已启用，您可以使用 Enter 键执行主要搜索（表格顶部高亮显示）。您还可以为此搜索设置其他快捷键。

常见问题解答

如何添加特定搜索引擎？

最简单的方法是右键单击搜索引擎的输入框，然后选择“添加到搜索栏...”。或者，您也可以通过“自定义搜索”下的高级选项手动添加自定义搜索。

如何在新的标签页中打开搜索结果？

只需使用鼠标中键或按住 Ctrl 键单击搜索按钮（就像单击普通链接一样）。要通过左键单击在新的标签页中打开搜索结果，请选中“默认在新标签页中打开搜索结果”复选框。

您也可以在按下快捷键的同时按住 Ctrl 键。或者，您可以通过“自定义搜索”下的高级选项指定自己的新标签页快捷键。

对于您的主搜索（最左侧的搜索，在自定义搜索顶部高亮显示），您还可以按 Ctrl+Enter。

如何使搜索栏像永久工具栏一样工作？

您可以考虑以下“高级设置”中的一些选项：启用“在浏览会话开始时显示搜索栏”；禁用“转到新页面时隐藏搜索栏”；禁用“在每个页面上独立显示和隐藏搜索栏”。并启用“搜索栏会遮挡网页”。

很遗憾，由于所有 Chrome 扩展程序都限制为仅使用一个工具栏按钮，因此无法将搜索栏显示在地址栏旁边。

如何备份或将我的设置传输到另一台计算机？

您可以使用“高级设置”下的“导入/导出设置”功能来完成此操作。对于手动导入/导出，请复制框中的文本并将其保存到方便的位置。然后，要恢复您的设置，请将文本粘贴到框中并单击“保存更改”。对于 Chrome 同步，请先确保您已登录 Chrome 并启用同步功能。然后使用按钮将您的设置保存到您的 Google 帐户并从中恢复。

如果您的设置占用空间过大，这些功能可能无法正常工作。

最常见的原因是上传了过大的图片。

如何同时执行两个不同的搜索？

您可以将一个自定义搜索的快捷键设置为与另一个自定义搜索的新标签页快捷键相同。这可以在“自定义搜索”下的高级选项中进行设置。

如何高亮显示由多个单词组成的精确短语？

只需将短语放在引号内即可。

如何在新标签页中使用显示搜索栏的热键？

搜索栏的热键在新标签页或 Chrome 设置页面可能无法使用。不过，您可以从 Chrome 扩展程序页面的菜单中选择“键盘快捷键”来设置其他热键。

为什么我按下工具栏按钮时，搜索栏有时会以弹出窗口的形式出现？

完整版的搜索栏使用内容脚本，因此无法在新标签页或 Chrome 设置页面（例如此页面）上运行。它也无法在您于“高级设置”下的“阻止搜索栏在这些主机名上运行（弹出窗口除外）”中指定的任何主机名上运行。“弹出窗口”版本允许您通过工具栏按钮从这些页面访问搜索栏的主要功能。

但是，与网页交互的功能，例如搜索选定文本和高亮显示搜索词，已被禁用。

在“高级设置”中，有一个选项可以在按下工具栏按钮时始终以弹出窗口的形式显示搜索栏。有时，即使您未选择此选项，并且在已允许搜索栏运行的普通网页上，弹出窗口也会出现。最常见的原因是页面尚未加载完成。

为什么我按下工具栏按钮时有时会出现一个空白的小提示？

最常见的原因是搜索栏仍在加载。首次打开搜索栏时，加载时间可能会稍长一些。

请耐心等待。

为什么大多数“javascript:”URL 不再有效？

以前，搜索栏允许使用 javascript: 协议的主页 URL 和搜索 URL。但是，从 Manifest V3 开始，Chrome 扩展程序不再允许执行用户提供的任意 JavaScript 代码。这是为了提高安全性，详情请参阅此处。

因此，除了少数特殊 URL（例如 `javascript:document.execCommand("copy")`）会触发剪贴板复制操作外，带有 `javascript:` 协议的首页 URL 和搜索 URL 在搜索栏中不再有效。

由于此限制，搜索栏现在支持在搜索 URL 和首页 URL 中使用高级的 `%{<JavaScript expression>}` 语法，例如 `%{u.replace(/\/$/,"").replace(/\/[^\/]*$/,"").replace(/^[a-z-]*:\/$/,u)}` 和 `https://chrome.google.com/webstore/search/%{encodeURIComponent(s)}`。此语法在“自定义搜索”下的高级选项说明中有更详细的解释。

为什么 SearchBar 需要“访问我在所有网站上的数据”？

Chrome 需要请求此权限才能将 SearchBar 插入到每个页面。但是，SearchBar 不会在其声明的功能之外更改或使用您的任何数据。

更多信息，请参阅 SearchBar 的隐私权政策，您可以此处查看。

如何重置为默认设置？

您可以使用“高级设置”下的“手动导入/导出”功能，删除所有内容并点击“保存更改”即可。

是什么启发您创建了如此出色的扩展程序？

这款扩展程序源于我对 Chrome 默认搜索系统的个人不满，以及对类似扩展程序功能的失望。我希望它能像改善我的浏览体验一样改善您的浏览体验，也期待您的反馈。
